# Kanban Board

## Qué es

Un tablero visual donde ves qué están haciendo tus agentes en tiempo real.

## Quick path

```bash
# 1. Abrir
ywai kanban

# 2. Ver en navegador
http://localhost:5768
```

## Las columnas

```
Backlog → Ready → In Progress → Review → Done
```

| Columna | Significado |
|---|---|
| **Backlog** | Tarea creada, no iniciada |
| **Ready** | Lista para empezar |
| **In Progress** | Agente trabajando |
| **Review** | Esperando tu aprobación |
| **Done** | Completada |

## Qué podés hacer

| Acción | Cómo |
|---|---|
| **Ver progreso** | Click en una card |
| **Responder preguntas** | Click → escribir respuesta |
| **Redirigir agente** | Click → "Steer" → escribir instrucción |
| **Detener agente** | Click → "Stop" |

## Ejemplo

```
Tú: "Implementa auth con JWT"
     ↓
Orchestrator crea cards:
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ 🟢 architect│  │ 🔵 dev      │  │ 🟡 qa       │
│ Diseña      │  │ Implementa  │  │ Testea      │
└─────────────┘  └─────────────┘  └─────────────┘
     ↓
En el Kanban ves las cards moverse de izquierda a derecha
```

## Cuándo usar Kanban

| Situación | Kanban |
|---|---|
| Tarea compleja con múltiples agentes | ✅ |
| Querés monitorear progreso | ✅ |
| Tarea simple de un solo agente | No necesitás |

<Aside type="tip">
El Kanban es parte del servidor de control. Cuando ejecutás `ywai kanban`, también tenés acceso a Workflow Studio y Settings.
</Aside>
