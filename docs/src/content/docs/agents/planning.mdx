# Planning Agent

## Qué hace

Investiga el codebase, analiza lo que se necesita, y crea un plan estructurado.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Planificar feature | `@planning Plan the implementation of RBAC` |
| Analizar impacto | `@planning What would break if we change auth?` |
| Crear plan de trabajo | `@planning Break down this epic into tasks` |

## Lo que hace

1. **Investiga** — Usa `@finder` y `@ask`
2. **Analiza** — Identifica archivos a cambiar
3. **Planifica** — Crea plan paso a paso
4. **Documenta** — Escribe plan estructurado

## Ejemplo de plan

```markdown
## Plan: RBAC Implementation

### Subtareas
1. Crear modelo de datos (roles, permissions)
2. Implementar middleware de autorización
3. Crear endpoints de gestión
4. Tests de integración

### Dependencias
- Tarea 2 depende de 1
- Tarea 3 depende de 1
- Tarea 4 depende de 2, 3
```

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Investigar codebase | Implementar código (`@dev`) |
| Crear planes | Decisiones de arquitectura (`@architect`) |
| Documentar | Escribir tests (`@qa`) |

<Aside type="tip">
Un buen plan ahorra horas de implementación incorrecta. Usá `@planning` antes de features grandes.
</Aside>
