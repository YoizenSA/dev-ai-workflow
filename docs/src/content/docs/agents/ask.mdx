# Ask Agent

## Qué hace

Responde preguntas, explica código, investiga tecnologías.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Pregunta técnica | `@ask ¿Cómo funciona JWT refresh token rotation?` |
| Explicar código | `@ask Explica el middleware de auth` |
| Comparar opciones | `@ask ¿Zustand o Jotai para estado?` |
| Investigar API | `@ask ¿Cómo uso Stripe checkout sessions?` |

## Cuándo delegar vs manejar directo

| Maneja directo | Delega |
|---|---|
| Pregunta con una respuesta | Requiere código → `@dev` |
| Comparación | Decisión de diseño → `@architect` |
| Explicar código existente | Trabajo multi-agente → orchestrator |

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Responder preguntas | Implementar código (`@dev`) |
| Explicar código | Escribir tests (`@qa`) |
| Investigar tecnologías | Decisiones de diseño (`@architect`) |

<Aside type="tip">
`@ask` es tu punto de partida cuando no sabés por dónde empezar.
</Aside>
