# QA Orchestrator

## Qué hace

Coordina el equipo de automatización de testing. Divide el trabajo entre agentes especializados.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Testing completo | "Testea todo el flujo de checkout" |
| Estrategia | "Diseña estrategia para el módulo de pagos" |

## Flujo

1. `@qa-analyst` diseña estrategia
2. `@qa-finder` busca código relacionado
3. `@qa-dev` escribe tests
4. `@qa-reviewer` revisa calidad

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Coordinar equipo | Escribir tests (`@qa-dev`) |
| Delegar tareas | Revisar tests (`@qa-reviewer`) |
| Ensamblar resultados | Diseñar estrategia (`@qa-analyst`) |

<Aside type="tip">
Para testing simple, usá `@qa` directamente. El qa-orchestrator es para estrategias completas.
</Aside>
