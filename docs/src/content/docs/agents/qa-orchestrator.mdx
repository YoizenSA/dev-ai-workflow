# QA Orchestrator

Coordina el **equipo de automatización de testing**. Divide el trabajo de testing entre agentes especializados y ensambla los resultados.

<Badge text="Grupo: qa-automation" variant="caution" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Testing completo | "Test the entire checkout flow" |
| Estrategia de testing | "Design a test strategy for the new feature" |
| Múltiples tipos de test | "Unit + integration + e2e for the auth module" |

## Lo que hace

1. **Analiza** qué necesita testing (usa `@qa-analyst`)
2. **Busca** código relevante (usa `@qa-finder`)
3. **Delega** escritura de tests (usa `@qa-dev`)
4. **Delega** revisión de tests (usa `@qa-reviewer`)
5. **Responde** preguntas de testing (usa `@qa-ask`)

## Agentes que coordina

| Agente | Rol |
|---|---|
| `@qa-analyst` | Diseña estrategia de testing |
| `@qa-finder` | Busca código relacionado |
| `@qa-dev` | Escribe tests automatizados |
| `@qa-reviewer` | Revisa calidad de tests |
| `@qa-ask` | Responde preguntas de testing |

<Aside type="tip">
Para testing simple, usa `@qa` directamente. El qa-orchestrator es para cuando necesitas múltiples tipos de test o una estrategia completa.
</Aside>
