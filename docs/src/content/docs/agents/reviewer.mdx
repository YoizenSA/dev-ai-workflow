# Reviewer Agent

## Qué hace

Revisa PRs, encuentra issues, y sugiere mejoras.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Code review | `@reviewer Revisa src/auth/` |
| Seguridad | `@reviewer Busca SQL injection en UserRepository` |
| Performance | `@reviewer ¿Hay N+1 queries en order list?` |
| Calidad | `@reviewer ¿El código sigue los patrones del proyecto?` |

## Niveles de severidad

| Nivel | Significado | Acción |
|---|---|---|
| 🔴 CRITICAL | Debe arreglarse | Antes de merge |
| 🟠 HIGH | Debería arreglarse | Pronto |
| 🟡 MEDIUM | Mejora recomendada | Opcional |
| 🟢 LOW | Estilo | Opcional |

## Qué revisa

- **Correctitud:** ¿Hace lo que dice? ¿Edge cases cubiertos?
- **Seguridad:** Inyección, XSS, CSRF, secretos hardcodeados
- **Performance:** N+1 queries, operaciones innecesarias
- **Mantenibilidad:** Legible, sin duplicado, sigue patrones

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Encontrar bugs | Implementar fixes (`@dev`) |
| Revisar seguridad | Escribir tests (`@qa`) |
| Sugerir mejoras | Decisiones de arquitectura (`@architect`) |

<Aside type="tip">
El reviewer es la última línea de defensa antes de merge. Usalo después de `@dev` y `@qa`.
</Aside>
