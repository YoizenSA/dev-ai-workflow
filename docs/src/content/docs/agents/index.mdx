# Agentes

## Quick reference

Usá `@nombre-del-agente` en opencode:

```
@dev Implementa el endpoint de login
@qa Escribe tests para UserService
@architect ¿Qué patrón usamos para auth?
@reviewer Revisa el PR de pagos
@devops Crea un Dockerfile
@ask ¿Cómo funciona el middleware actual?
```

## El equipo

### Core (10 agentes)

| Agente | Rol | Para qué |
|---|---|---|
| [**orchestrator**](/dev-ai-workflow/agents/orchestrator) | Coordinador | Coordina múltiples agentes para tareas complejas |
| [**dev**](/dev-ai-workflow/agents/dev) | Desarrollador | Escribir código, arreglar bugs, refactorizar |
| [**qa**](/dev-ai-workflow/agents/qa) | QA Engineer | Tests, cobertura, validación |
| [**architect**](/dev-ai-workflow/agents/architect) | Arquitecto | Decisiones de diseño, patrones, tradeoffs |
| [**reviewer**](/dev-ai-workflow/agents/reviewer) | Revisor | Code review, bugs, seguridad |
| [**devops**](/dev-ai-workflow/agents/devops) | DevOps | CI/CD, Docker, K8s, despliegues |
| [**ask**](/dev-ai-workflow/agents/ask) | Investigador | Preguntas, análisis, explicaciones |
| [**finder**](/dev-ai-workflow/agents/finder) | Explorador | Buscar archivos, entender estructura |
| [**planning**](/dev-ai-workflow/agents/planning) | Planificador | Investigar y planificar antes de implementar |
| [**memory**](/dev-ai-workflow/agents/memory) | Archivista | Memoria persistente entre sesiones |

## Cuándo usar cada agente

| Necesidad | Agente |
|---|---|
| Sola pregunta | `@ask` |
| Necesitás código | `@dev` |
| Necesitás tests | `@qa` |
| Decisión de diseño | `@architect` |
| Revisar código | `@reviewer` |
| Infraestructura | `@devops` |
| Buscar código | `@finder` |
| Planificar | `@planning` |
| Recordar decisiones | `@memory` |
| Tarea compleja | Orchestrator (automático) |

<Aside type="tip">
No necesitás invocar agentes uno por uno. Decile al orchestrator qué necesitás y él distribuye el trabajo.
</Aside>
