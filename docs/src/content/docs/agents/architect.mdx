# Architect Agent

## Qué hace

Evalúa opciones de diseño, propone soluciones, y documenta decisiones.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Decisión de diseño | `@architect ¿WebSockets o SSE para notificaciones?` |
| Evaluar patrón | `@architect ¿CQRS es apropiado aquí?` |
| Análisis de tradeoffs | `@architect ¿Monolith o microservices?` |
| Revisar arquitectura | `@architect Revisa la estructura actual de auth` |

## Lo que hace

1. Evalúa el código existente
2. Identifica oportunidades de mejora
3. Propone soluciones con tradeoffs
4. Documenta decisiones (ADRs)
5. Delega implementación a `@dev`

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Evaluar opciones de diseño | Implementar código (`@dev`) |
| Proponer soluciones | Revisar PRs por estilo (`@reviewer`) |
| Documentar decisiones | Configurar infraestructura (`@devops`) |

<Aside type="tip">
Usá `@architect` antes de features grandes. Un buen plan ahorra horas de implementación incorrecta.
</Aside>
