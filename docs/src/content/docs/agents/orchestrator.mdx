# Orchestrator Agent

## Qué hace

Coordina múltiples agentes para tareas complejas. Divide el trabajo, delega, y ensambla resultados.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Feature compleja | "Implementa sistema de pagos" |
| Proyecto nuevo | "Crea la estructura base" |
| Refactoring grande | "Refactoriza toda la capa de datos" |

## Lo que hace

1. **Investiga** — Usa `@finder`, `@ask`
2. **Planifica** — Divide en subtareas
3. **Delega** — Asigna al agente correcto
4. **Coordina** — Maneja dependencias
5. **Ensambla** — Integra resultados

## Flujo

```
Tarea compleja → Orchestrator investiga → Planifica → Delega
                    ↓
         ┌─────────┬─────────┬─────────┐
         │@architect│  @dev   │   @qa   │
         │ diseña   │implementa│  testea │
         └─────────┴─────────┴─────────┘
                    ↓
              @reviewer revisa → Resultado
```

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Coordinar agentes | Escribir código (`@dev`) |
| Planificar subtareas | Escribir tests (`@qa`) |
| Manejar dependencias | Decisiones de diseño (`@architect`) |

<Aside type="tip">
No necesitás invocar al orchestrator directamente. opencode lo usa automáticamente cuando la tarea es compleja.
</Aside>
