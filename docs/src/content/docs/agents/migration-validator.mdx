# Migration Validator

## Qué hace

Valida que la migración mantenga paridad funcional — compara comportamiento antes y después.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Validar migración | "Verify the migrated login page works the same" |
| Comparar comportamiento | "Does the new API return the same responses?" |

## Qué verifica

- UI: componentes renderizan igual
- API: responses con mismo formato
- Data: datos se persisten correctamente
- Accessibility: no se pierde accesibilidad

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Comparar comportamiento | Modificar código |
| Documentar diferencias | Arreglar findings (`@dev`) |
| Reportar con severidad | Aprobar sin evidencia |
