# QA Ask

Responde **preguntas sobre testing** — explica patrones, frameworks, y mejores prácticas.

<Badge text="Grupo: qa-automation" variant="caution" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Preguntas de testing | "How do I mock API calls in Vitest?" |
| Explicar frameworks | "What's the difference between unit and integration tests?" |
| Mejores prácticas | "How should I test error handling?" |

## Lo que hace

1. **Responde** preguntas sobre testing
2. **Explica** frameworks y herramientas
3. **Sugiere** mejores prácticas
4. **Da** ejemplos de código

## Lo que NO hace

- ❌ NO escribe tests completos (eso es `@qa-dev`)
- ❌ NO revisa código (eso es `@qa-reviewer`)
- ❌ NO explora codebase (eso es `@qa-finder`)
