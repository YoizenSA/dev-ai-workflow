# QA Dev

## Qué hace

Escribe tests automatizados — unit, integration, y e2e.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Tests unitarios | `@qa-dev Escribe tests para UserService` |
| Tests integration | `@qa-dev Tests de integración para la API` |
| Tests E2E | `@qa-dev Playwright tests para checkout` |

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Escribir tests | Explorar codebase (`@qa-finder`) |
| Cubrir edge cases | Revisar su código (`@qa-reviewer`) |
| Verificar tests pasan | Diseñar estrategia (`@qa-analyst`) |
