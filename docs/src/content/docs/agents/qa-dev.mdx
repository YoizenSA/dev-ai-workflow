# QA Dev

Escribe **tests automatizados** — unit, integration, y e2e. Implementa la estrategia diseñada por `@qa-analyst`.

<Badge text="Grupo: qa-automation" variant="caution" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Tests unitarios | "Write unit tests for UserService" |
| Tests de integración | "Write integration tests for the API" |
| Tests E2E | "Write Playwright tests for the checkout flow" |

## Lo que hace

1. **Lee** la estrategia de `@qa-analyst`
2. **Escribe** tests siguiendo TDD (red → green → refactor)
3. **Cubre** casos normales, edge cases, y errores
4. **Verifica** que los tests pasan

## Lo que NO hace

- ❌ NO explora codebase (eso es `@qa-finder`)
- ❌ NO revisa su propio código (eso es `@qa-reviewer`)
- ❌ NO diseña estrategia (eso es `@qa-analyst`)
