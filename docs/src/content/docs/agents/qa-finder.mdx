# QA Finder

Busca **código relacionado con testing** — encuentra archivos, patrones de test existentes, y código que necesita cobertura.

<Badge text="Grupo: qa-automation" variant="caution" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Buscar tests existentes | "Find all tests related to auth" |
| Encontrar código sin cobertura | "What code has no tests?" |
| Explorar patrones | "How do we write tests in this project?" |

## Lo que hace

1. **Busca** archivos de test existentes
2. **Identifica** código sin cobertura
3. **Encuentra** patrones de test del proyecto
4. **Reporta** el estado de testing

## Lo que NO hace

- ❌ NO escribe tests (eso es `@qa-dev`)
- ❌ NO revisa tests (eso es `@qa-reviewer`)
- ❌ NO diseña estrategia (eso es `@qa-analyst`)
