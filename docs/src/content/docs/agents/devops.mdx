# DevOps Agent

El agente devops se especializa en **infraestructura, CI/CD, contenedores y despliegues**. Configura pipelines, crea Dockerfiles y gestiona despliegues.

<Badge text="Rol: DevOps Engineer" variant="note" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| CI/CD pipelines | "Set up GitHub Actions for this project" |
| Docker | "Create a Dockerfile for the Node.js service" |
| Kubernetes | "Write K8s manifests for the API" |
| Despliegues | "Set up staging environment on AWS" |
| Monitoreo | "Add health checks and logging" |

## Lo que hace

1. **Configura** pipelines de CI/CD (GitHub Actions, Azure Pipelines, etc.)
2. **Crea** Dockerfiles optimizados y seguros
3. **Escribe** manifiestos de Kubernetes (Deployments, Services, Ingress)
4. **Configura** Helm charts para despliegues complejos
5. **Automatiza** procesos de build, test y deploy

## Lo que NO hace

- ❌ NO implementa features de aplicación (eso es `@dev`)
- ❌ NO diseña arquitectura de aplicación (eso es `@architect`)
- ❌ NO escribe tests de aplicación (eso es `@qa`)
- ❌ NO revisa código de aplicación (eso es `@reviewer`)

## Ejemplo de uso

```
@devops Create a production-ready Dockerfile for our Next.js app.
Include: multi-stage build, non-root user, health check, and 
optimized layer caching. Also create a docker-compose.yml for 
local development with PostgreSQL and Redis.
```

<Aside type="tip">
El agente devops es ideal para configurar el pipeline completo de tu proyecto, desde el push hasta producción.
</Aside>

## Flujo típico

```
@architect (diseña infra) → @devops (configura) → @dev (integra) → @qa (verifica)
```

## Skills que usa

| Skill | Cuándo |
|---|---|
| `devops` | Azure Pipelines, Helm, K8s |
| `docker` | Dockerfiles, compose, hardening |
