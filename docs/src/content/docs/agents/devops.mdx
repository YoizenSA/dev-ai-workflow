# DevOps Agent

## Qué hace

Configura pipelines, crea Dockerfiles, y gestiona despliegues.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| CI/CD | `@devops Configura GitHub Actions` |
| Docker | `@devops Crea un Dockerfile para Node.js` |
| Kubernetes | `@devops Escribe K8s manifests para la API` |
| Deploy | `@devops Configura deploy a staging` |

## Lo que hace

1. Configura pipelines de CI/CD
2. Crea Dockerfiles optimizados
3. Escribe manifiestos de Kubernetes
4. Crea Helm charts
5. Automatiza build, test, deploy

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Configurar CI/CD | Implementar features (`@dev`) |
| Crear Dockerfiles | Diseñar arquitectura (`@architect`) |
| Escribir K8s manifests | Escribir tests (`@qa`) |

<Aside type="tip">
Para un Dockerfile simple, `@devops` solo es suficiente. Para pipelines completos, describí todos los requerimientos de una vez.
</Aside>
