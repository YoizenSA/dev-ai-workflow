# QA Analyst

Diseña **estrategias de testing** y analiza cobertura. Decide qué testear, cómo, y con qué prioridad.

<Badge text="Grupo: qa-automation" variant="caution" />

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Estrategia de testing | "Design a test strategy for the payment module" |
| Análisis de cobertura | "What's our test coverage for the auth flow?" |
| Priorización | "Which tests should we write first?" |

## Lo que hace

1. **Analiza** el código y los requisitos
2. **Diseña** la estrategia de testing
3. **Prioriza** qué testear primero
4. **Identifica** gaps de cobertura

## Lo que NO hace

- ❌ NO escribe tests automatizados (eso es `@qa-dev`)
- ❌ NO explora codebase (eso es `@qa-finder`)
- ❌ NO revisa código de tests (eso es `@qa-reviewer`)
