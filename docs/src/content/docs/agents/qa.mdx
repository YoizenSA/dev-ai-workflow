# QA Agent

## Qué hace

Diseña estrategia de testing, escribe tests, y valida implementaciones.

## Cuándo usarlo

| Tarea | Ejemplo |
|---|---|
| Escribir tests | `@qa Escribe tests para UserService` |
| Estrategia | `@qa Diseña estrategia para el flujo de pagos` |
| Validar | `@qa Verifica que el login funcione` |
| Cobertura | `@qa ¿Qué cobertura tenemos en auth?` |

## Tipos de tests

| Tipo | Framework | Ejemplo |
|---|---|---|
| Unit | Jest / Vitest / Go testing | `test('calcula total con impuesto')` |
| Integration | Supertest / httpexpect | `test('POST /login retorna JWT')` |
| E2E | Playwright | `test('usuario completa checkout')` |

## Convenciones de nombres

```
✅ "throws ValidationError when email is invalid"
✅ "calculates total with tax for items above threshold"
❌ "works"
❌ "test1"
```

## Principios

- Un test, una asertación lógica
- Tests independientes (no dependen de orden)
- Tests rápidos (unit < 100ms, integration < 1s)
- Tests determinísticos (mismo resultado siempre)

## Límites

| ✅ Puede | ❌ No puede |
|---|---|
| Diseñar estrategia de testing | Implementar features (`@dev`) |
| Escribir tests | Revisar por seguridad (`@reviewer`) |
| Analizar cobertura | Configurar CI/CD (`@devops`) |

<Aside type="tip">
Los tests son documentación viva. Deben explicar qué hace el código, no cómo lo hace.
</Aside>
