# Guías

<Aside type="tip">
**Patrón común:** Vos le decís al orchestrator qué necesitás → él distribuye a los agentes correctos → te reporta el resultado.
</Aside>

## Las 7 guías

| Guía | Qué hacés | Qué pasa |
|---|---|---|
| [**Implementar Feature**](/guides/feature) | "Implementa auth con JWT" | architect → dev → qa → reviewer |
| [**Arreglar Bug**](/guides/bugfix) | "El login falla con +" | dev investiga → arregla → qa testea |
| [**Code Review**](/guides/review) | "Revisa src/auth/" | reviewer revisa → reporta findings |
| [**Migración**](/guides/migration) | "Migra auth de Angular a React" | migration-scope → planner → dev → validator |
| [**Testing**](/guides/testing) | "Testea el checkout" | qa-analyst → finder → dev → reviewer |
| [**CI/CD**](/guides/cicd) | "Configura GitHub Actions" | devops configura pipeline |
| [**Refactoring**](/guides/refactoring) | "Usá repository pattern" | architect analiza → dev refactoriza → qa verifica |

## Ejemplo rápido

```
Vos: "Implementa un sistema de pagos con Stripe"

Orchestrator:
  → @architect: diseña PaymentService
  → @dev: implementa
  → @qa: tests
  → @reviewer: revisa por seguridad
  
Resultado: "Feature completa. 2 issues corregidos."
```

## Cuándo usar cada guía

| Situación | Guía |
|---|---|
| Feature nueva | [Implementar Feature](/guides/feature) |
| Algo está roto | [Arreglar Bug](/guides/bugfix) |
| Antes de mergear | [Code Review](/guides/review) |
| Cambiar de framework | [Migración](/guides/migration) |
| Necesitás cobertura | [Testing](/guides/testing) |
| Deploy a producción | [CI/CD](/guides/cicd) |
| Mejorar código existente | [Refactoring](/guides/refactoring) |
