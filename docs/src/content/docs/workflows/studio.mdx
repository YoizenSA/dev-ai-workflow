# Workflow Studio

## Qué es

Un editor visual donde diseñás flujos de trabajo en un canvas y los exportás a opencode.

## Quick path

```bash
# 1. Abrir
ywai kanban → http://localhost:5768/workflows

# 2. Diseñar
Arrastrar nodos → Conectar → Configurar

# 3. Exportar
Click "Export" → Se crea el command en opencode
```

## Tipos de nodos

| Nodo | Qué hace | Ejemplo |
|---|---|---|
| **SubAgent** | Delega a un agente | Dev, QA, Architect |
| **Prompt** | Instrucción fija | "Usa TypeScript estricto" |
| **If/Else** | Decisión condicional | "Si hay errores, ir a Dev" |
| **AskUserQuestion** | Pregunta al usuario | "¿Mergeamos a main?" |
| **Skill** | Aplica un skill | react-19, tailwind-4 |

## Ejemplo

```
[Start] → [Architect] → [Dev] → [QA] → [If: Tests OK?]
                                          ├─ Sí → [Reviewer] → [End]
                                          └─ No → [Dev] → [QA]
```

## Qué se exporta

| Workflow element | opencode primitive |
|---|---|
| Workflow completo | Slash command (`/nombre`) |
| Orchestrator | Agent (`nombre-orchestrator.md`) |
| Nodos SubAgent | Agents (`nombre-agente.md`) |

## Validación

El sistema verifica:
- ✅ Nodo de inicio y final
- ✅ Sin ciclos
- ✅ Campos requeridos completos

<Aside type="tip">
Studio es ideal para flujos recurrentes. Diseñá una vez, exportá, y todo el equipo lo usa como command.
</Aside>
