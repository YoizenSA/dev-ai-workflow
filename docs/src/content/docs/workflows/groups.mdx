# Agent Groups

## Qué son

Equipos predefinidos de agentes diseñados para trabajar juntos en tareas específicas.

## Los 3 grupos

### Core

| Agente | Rol |
|---|---|
| `orchestrator` | Coordinador |
| `dev` | Implementa código |
| `qa` | Escribe tests |
| `architect` | Decisiones de diseño |
| `reviewer` | Revisión de código |
| `devops` | Infraestructura |
| `ask` | Preguntas |
| `finder` | Búsqueda |
| `planning` | Planificación |
| `memory` | Memoria persistente |

**Para qué:** Desarrollo general.

### Social Refactor

| Agente | Rol |
|---|---|
| `migration-orchestrator` | Coordina migración |
| `migration-planner` | Planifica |
| `migration-scope` | Clasifica alcance |
| `migration-validator` | Valida paridad |
| `migration-validator-focused` | Valida una página |

**Para qué:** Migraciones entre frameworks.

### QA Automation

| Agente | Rol |
|---|---|
| `qa-orchestrator` | Coordina testing |
| `qa-analyst` | Estrategia |
| `qa-dev` | Escribe tests |
| `qa-finder` | Busca código |
| `qa-reviewer` | Revisa tests |
| `qa-ask` | Preguntas de testing |

**Para qué:** Testing completo con estrategia.

## Delegación entre agentes

```
Core:
  orchestrator → [dev, qa, architect, reviewer, devops, ask, finder, planning, memory]

Social Refactor:
  migration-orchestrator → [migration-planner, migration-scope, migration-validator, dev]

QA Automation:
  qa-orchestrator → [qa-analyst, qa-dev, qa-finder, qa-reviewer, qa-ask]
```

## Cuándo usar cada grupo

| Situación | Grupo |
|---|---|
| Feature nueva | Core |
| Bug fix | Core |
| Migrar framework | Social Refactor |
| Testing completo | QA Automation |

<Aside type="tip">
No necesitás invocar los grupos directamente. El orchestrator elige el grupo correcto según la tarea.
</Aside>
