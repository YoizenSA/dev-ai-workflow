## ¿Qué es?

Workflow Studio es un editor visual de flujos de trabajo multi-agente. Permite diseñar workflows en un canvas React Flow y exportarlos a primitivas nativas de opencode (comandos slash y agentes).

Está diseñado para equipos que quieren definir pipelines de desarrollo repetibles con roles, gates y condiciones, sin escribir configuraciones manuales.

## Dónde está

La interfaz está disponible en la ruta `/workflows` de la UI de control:

```
http://localhost:5768/workflows
```

## Cómo funciona

### 1. Diseña

Arrastra nodos desde la paleta al canvas y conéctalos. Los tipos de nodos disponibles son:

| Nodo | Descripción |
|------|-------------|
| **SubAgent** | Delega trabajo a un sub-agente (dev, qa, reviewer, etc.) |
| **AskUserQuestion** | Pausa el flujo para consultar al usuario |
| **Prompt** | Muestra un mensaje o instrucción |
| **If/Else** | Rama condicional basada en una condición |
| **Switch** | Múltiples ramas basadas en el valor de una expresión |
| **Skill** | Ejecuta una skill específica |
| **MCP** | Llama a una herramienta MCP |
| **Group** | Agrupa nodos visualmente |

### 2. Edita

Cada nodo tiene un panel lateral donde se configuran sus propiedades:

- **System prompt**: instrucciones base para el sub-agente
- **Task prompt**: la tarea específica a realizar
- **Tools**: herramientas disponibles para el nodo
- **Model**: modelo a utilizar (opcional)
- **Options**: opciones adicionales (tiempo límite, etc.)
- **Conditions**: condiciones para nodos If/Else y Switch

### 3. Valida

El grafo se valida automáticamente contra reglas estructurales:

- Un solo nodo de inicio y un solo nodo de fin
- Sin ciclos (el grafo debe ser acíclico)
- Todos los nodos alcanzables desde el inicio
- Límites de campos respetados

### 4. Exporta

El workflow se exporta a primitivas nativas de opencode:

| Elemento del workflow | Primitiva opencode | Salida |
|----------------------|--------------------|--------|
| Workflow completo (entry point) | Slash command | `~/.config/opencode/commands/<name>.md` |
| Orchestrator persona | Agent | `~/.config/opencode/agents/<name>-orchestrator.md` |
| Nodos `subAgent` | Agents | `~/.config/opencode/agents/<name>-<slug>.md` |
| `askUserQuestion` / `ifElse` / `switch` | Routing instructions | Embebido en el system prompt del orchestrator |

El comando slash se invoca como `/<nombre>` en opencode.

## Source of truth vs generated output

| | Source (editable) | Generated (lo que lee opencode) |
|---|---|---|
| Ruta | `~/.ywai/workflows/<name>.json` | `~/.config/opencode/{commands,agents}/` |
| Formato | JSON | Markdown + archivos de agente |
| Editable | Sí, con Workflow Studio | No — se regenera al exportar |

El formato JSON es estable y round-trip: puedes importar un workflow exportado, editarlo y re-exportarlo sin perder información. Si faltan nodos de inicio o fin, se añaden automáticamente al importar.

## Backend

El Workflow Studio está respaldado por:

- **Modelo**: `internal/workflows/` (modelo, store, validador, exporter, importer)
- **API REST**: `internal/control/workflows.go` (endpoints en `/api/workflows`)
- **Frontend**: React Flow en la UI de control
