# Workflows

## Los 3 componentes

| Componente | Qué es | Ver más |
|---|---|---|
| **Workflow Studio** | Editor visual en el navegador | [Studio →](/workflows/studio) |
| **Workflow Commands** | Slash commands generados | [Commands →](/workflows/commands) |
| **Agent Groups** | Equipos predefinidos de agentes | [Groups →](/workflows/groups) |

## Cómo se relacionan

```
Studio (diseñas) → JSON (guardas) → Commands (exportas) → Groups (agentes)
```

## Quick path

```bash
# 1. Abrir Studio
ywai kanban → http://localhost:5768/workflows

# 2. Diseñar flujo
Arrastrar nodos → Conectar → Configurar

# 3. Exportar
Click "Export" → Se crea /feature-mi-flujo

# 4. Usar
/feature-mi-flujo Agregar login con Google
```

## Commands preinstalados

| Command | Flujo | Para qué |
|---|---|---|
| `/feature-delivery` | planning → architect → dev → qa → reviewer | Features completas |
| `/social-refactor` | scope → planner → dev → validator | Migraciones |
| `/qa-automation` | analyst → finder → dev → reviewer | Testing completo |

## Cuándo usar cada componente

| Necesidad | Componente |
|---|---|
| Usar un flujo predefinido | Commands |
| Diseñar un flujo nuevo | Studio |
| Entender qué agentes hay | Agent Groups |

<Aside type="tip">
Para la mayoría de casos, los commands preinstalados son suficientes. Usá Studio solo cuando necesités un flujo personalizado.
</Aside>
