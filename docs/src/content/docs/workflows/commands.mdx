# Workflow Commands

## Qué son

Comandos slash que lanzan un flujo completo de agentes. Los escribís en opencode y el orchestrator coordina todo.

## Commands preinstalados

| Command | Flujo | Para qué |
|---|---|---|
| `/feature-delivery` | planning → architect → dev → qa → reviewer | Features completas |
| `/social-refactor` | scope → planner → dev → validator | Migraciones |
| `/qa-automation` | analyst → finder → dev → reviewer | Testing completo |

## Cómo usarlos

```
/feature-delivery Implementar autenticación con JWT
```

Esto lanza automáticamente:
1. `@planning` investiga y planifica
2. `@architect` diseña la solución
3. `@dev` implementa
4. `@qa` testea
5. `@reviewer` revisa

## Crear tus propios commands

### Con Studio (visual)

1. Abrir `http://localhost:5768/workflows`
2. Diseñar el flujo
3. Exportar como command

### Con JSON (avanzado)

```json
{
  "name": "mi-command",
  "nodes": [
    { "id": "start", "type": "start" },
    { "id": "dev", "type": "subAgent", "agent": "dev" },
    { "id": "qa", "type": "subAgent", "agent": "qa" },
    { "id": "end", "type": "end" }
  ],
  "edges": [
    { "source": "start", "target": "dev" },
    { "source": "dev", "target": "qa" },
    { "source": "qa", "target": "end" }
  ]
}
```

## Cuándo usar commands vs agentes

| Situación | Command | Agente directo |
|---|---|---|
| Feature completa | `/feature-delivery` | |
| Migración | `/social-refactor` | |
| Testing completo | `/qa-automation` | |
| Bug simple | | `@dev` |
| Pregunta rápida | | `@ask` |

<Aside type="tip">
Los commands son para flujos complejos. Para tareas simples, usá un agente directamente.
</Aside>
