# Configuración

Guía para configurar ywai y personalizar el comportamiento de tus agentes.

## Estructura de configuración

Cuando ejecutas `ywai install`, se crean estos archivos:

```
~/.config/opencode/
├── agents/                 # Perfiles de agentes
│   ├── dev.md
│   ├── qa.md
│   ├── architect.md
│   ├── reviewer.md
│   ├── devops.md
│   └── ask.md
├── skills/                 # Skills instalados
│   ├── typescript/
│   ├── react-19/
│   ├── tailwind-4/
│   └── ...
└── commands/               # Comandos slash
```

## Perfiles de agentes

Cada agente tiene un perfil con tres archivos:

### AGENT.md — System prompt

El archivo principal que define cómo se comporta el agente. Contiene:
- Rol y responsabilidades
- Principios de trabajo
- Límites (qué puede y qué no puede hacer)
- Formato de handoff al orquestador

**Ejemplo (dev):**
```markdown
# Dev Agent

You implement features, fix bugs, and refactor.
Read before write, make small atomic changes, 
follow existing patterns, and test every change.
```

### tools.json — Permisos de herramientas

Define qué herramientas puede usar el agente:
```json
{
  "read": true,
  "write": true,
  "bash": true,
  "delegate": false
}
```

### skills.txt — Skills vinculados

Lista de skills que el agente tiene disponibles:
```
typescript
react-19
tailwind-4
git-commit
```

## Personalizar agentes

### Modificar un agente existente

<Steps>

1. **Encuentra el perfil** en `~/.config/opencode/agents/`
2. **Edita el AGENT.md** para cambiar el comportamiento
3. **Edita tools.json** para cambiar permisos
4. **Edita skills.txt** para agregar o quitar skills
5. **Reinicia opencode** para que tome los cambios

</Steps>

### Crear un agente nuevo

<Steps>

1. **Copia un perfil existente** como base
2. **Renombra** el archivo (ej: `mi-agente.md`)
3. **Edita** el system prompt para el nuevo rol
4. **Configura** permisos y skills
5. **Reinicia opencode**

</Steps>

## Configuración del proyecto

### AGENTS.md en el proyecto

Puedes crear un `AGENTS.md` en la raíz de tu proyecto para dar contexto específico:

```markdown
# AGENTS.md

## Convenciones del proyecto
- Usar TypeScript estricto
- Commits en formato convencional
- Tests con Vitest

## Estructura
- src/components/ — Componentes React
- src/services/ — Lógica de negocio
- src/utils/ — Utilidades compartidas

## Stack
- Next.js 14
- PostgreSQL
- Prisma ORM
- Tailwind CSS
```

Los agentes leen este archivo para entender tu proyecto.

### Skills personalizados del proyecto

Puedes crear skills específicos para tu proyecto:

```
mi-proyecto/
├── .agents/
│   └── skills/
│       └── mi-patron/
│           └── SKILL.md
└── ...
```

Luego registra con:
```bash
ywai skill-registry --cwd /mi/proyecto
```

## Verificar la configuración

```bash
ywai doctor
```

Esto verifica:
- ✅ gentle-ai instalado correctamente
- ✅ Skills enlazados
- ✅ Rutas válidas
- ✅ Sin conflictos
